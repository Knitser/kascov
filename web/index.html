      all smart coins

      script decoder

      paste any Kaspa script as hex and read it as opcodes — including the post-Toccata set: transaction introspection (KIP-17), covenant ops (KIP-20), zk verification (KIP-16). decoding happens in your browser; nothing leaves the page.

      try an example:
      what the colors mean
      • push data pushed onto the stack
      • standard the classic script opcodes (checks, math, stack)
      • introspection KIP-17 — the script reads its own transaction
      • covenant KIP-20 — smart-coin identity & state rules
      • zk KIP-16 — zero-knowledge proof verification
      • unknown not in the post-Toccata opcode table

      for developers

      kascov indexes every covenant on Kaspa and keeps the history nodes prune. all of it is yours as plain JSON — CORS is on, no keys, no rate cards. build something.

      the JSON API

      GET /data/testnet-10-live.json

      the small fast feed — stats, chain tip, and the ~150 newest life events. refreshes every few seconds; poll this.

      GET /data/testnet-10.json

      the grid — stats plus one summary row per covenant (id, status, balances, event count, born/last-active DAA). one request tells you about every smart coin on the network.

      GET /data/testnet-10/c/<covenant-id>.json

      one covenant's whole life — the complete event timeline and every UTXO with decoded scripts, recognized templates, and spend-time reveals.

      GET /data/testnet-10/tx/<txid>.json

      which smart coin did this transaction touch? answers {"covenant_id": …} or 404 — genesis, moves, and burns all resolve.

      GET /data/testnet-10/digest.json

      the last 24 hours in one object — births, moves, burns, value born (sompi), the busiest coin, the biggest birth, and how many coins are alive right now. refreshes about once a minute.

      GET /data/testnet-10/addr/<address-or-pubkey>.json

      which smart coins has this address touched? accepts kaspa:… / kaspatest:… or raw 32/33-byte pubkey hex. answers the canonical address, the pubkey, and grid-style rows with role hints (controls_now, states_seen). matches p2pk covenant states only.

      GET /data/testnet-10/templates.json

      what runs on this network — recognized script templates aggregated over every state UTXO: live_states / live_value now, ever_seen all-time, distinct covenants, and revealed_runs (compiled contracts proven at spend time; a tx sweeping N states counts N). unrecognized scripts are counted, not hidden.

      GET /data/testnet-10/activity.json?range=24h

      the histogram behind the activity chart — life-event counts (births / moves / burns) per DAA bucket. range is 1h | 6h | 24h | 48h | all (default 24h); bucket_daa is the bucket width and each bucket's daa its start. buckets with nothing in them are omitted — treat them as zeros. refreshes about every 30s.

      GET /data/testnet-10/stream

      server-sent events — one JSON message per life event the moment the indexer sees it (covenant_id, kind, txid, accepting_daa), plus a : ka comment every 25s. treat messages as hints and confirm through the polled feeds — some CDNs buffer streams, so keep a poll running.

      GET /data/mainnet-live.json · /data/mainnet.json · /data/mainnet/…

      same endpoints for mainnet — its first smart coins arrived July 2, 2026, and their complete history starts here. spend-time reveals and KIP-20-validated genesis exist for activity from that evening onward.

      base URL: https://kascov-explorer.web.app · Access-Control-Allow-Origin: * · served by an always-on indexer that follows the chain live

      try it

      # the pulse of the testnet, one request
curl -s https://kascov-explorer.web.app/data/testnet-10-live.json | jq .stats

# coins born in the last few minutes
curl -s https://kascov-explorer.web.app/data/testnet-10-live.json \
  | jq '.recent_events[] | select(.kind == "genesis")'

# the ten richest living smart coins
curl -s https://kascov-explorer.web.app/data/testnet-10.json \
  | jq '[.covenants[] | select(.status == "active")] | sort_by(-.live_value)[:10]'

# one covenant's whole life, scripts decoded
curl -s https://kascov-explorer.web.app/data/testnet-10/c/<covenant-id>.json | jq .events

# which coin did a transaction touch?
curl -s https://kascov-explorer.web.app/data/testnet-10/tx/<txid>.json

# the last 24 hours, one object
curl -s https://kascov-explorer.web.app/data/testnet-10/digest.json | jq

# every smart coin an address has touched (p2pk states)
curl -s https://kascov-explorer.web.app/data/testnet-10/addr/<address-or-pubkey>.json | jq .covenants

# what runs on this network, by recognized template
curl -s https://kascov-explorer.web.app/data/testnet-10/templates.json | jq .templates

# the activity histogram behind the chart
curl -s 'https://kascov-explorer.web.app/data/testnet-10/activity.json?range=24h' | jq .buckets

# push, not poll — every life event as it happens (SSE)
curl -N https://kascov-explorer.web.app/data/testnet-10/stream

      field dictionary

      covenant_id
      the smart coin's permanent 32-byte identity (KIP-20) — survives every move
      status
      active (has live UTXOs) or burned (retired)
      events[]
      the life story (per-coin endpoint): kind is genesis | transition | burn, ordered by seq; txid is the transaction, accepting_daa when the chain accepted it
      born_value
      grid rows only — what the coin held at birth (sompi)
      accepting_daa / DAA
      Kaspa's block-height-like clock — it ticks ~10 per second. tip_daa + tip_at_ms in each snapshot anchor it to wall-clock time
      processed_daa
      live + grid feeds — the DAA of the last chain block the indexer has actually applied. tip_daa − processed_daa is the honest sync lag: the indexer replays every block in order, so during a catch-up the tip runs ahead while nothing is skipped. null from indexes that haven't recorded it yet
      genesis_txid, genesis_daa
      where the coin was born — null if it was first seen mid-life (see lineage_complete)
      lineage_complete
      false means kascov started watching after the coin was born; earlier history is honestly missing
      utxos[]
      every state UTXO ever seen: outpoint, value (sompi, 1e8 = 1 KAS), live, script_hex + script_asm (pre-disassembled), uses_covenant_ops, uses_zk_ops; spent ones add spent_txid
      revealed_hex / revealed_asm
      spend-time decoding — when a P2SH state is spent, the unlocking script reveals the program it actually ran, verified against the committed blake2b hash (plus revealed_uses_covenant_ops/_zk_ops; non-P2SH spends carry sig_hex instead)
      template / state_fields
      recognized script shapes with labeled fields — p2pk state (owner_pubkey), p2sh commitment (program_hash), and compiled SilverScript contracts (Mecenas, Escrow, LastWill) once seen; revealed programs get revealed_template/revealed_fields
      templates[] / unrecognized
      templates endpoint — per script shape: live_states/live_value held now, ever_seen state UTXOs all-time, covenants distinct coins, revealed_runs spends whose verified P2SH reveal matched the template; unrecognized buckets everything else
      controls_now / states_seen
      address endpoint — true while the key still owns a live state of that coin (vs a past owner only), plus how many of the coin's states have carried the key; first_seen_daa/last_seen_daa bracket the relationship
      events[].payload / spent_budget
      v1 extras, captured per event/spend: the transaction payload (hex when ≤512 B, else payload_len) and the spending input's compute-budget commitment
      live_value, live_utxos
      what the coin holds right now
      events_truncated
      the full snapshot caps events per covenant (default 500); true means the tail was cut
      generated_at_ms
      when this JSON was built — your freshness check

      run kascov yourself

      the whole indexer is a small open-source Rust workspace. your own node, your own index, your own rules.

      git clone https://github.com/Knitser/kascov && cd kascov

# follow testnet-10 live (public resolver, zero setup)
cargo run -rp kascov -- --network testnet-10 sync --follow

# print covenant events as they happen (great for testing your own covenant)
cargo run -rp kascov -- --network testnet-10 watch

# machine-readable event stream
cargo run -rp kascov -- --network testnet-10 --json watch

# export the snapshots this site serves
cargo run -rp kascov -- --network testnet-10 export

      make your own smart coin on testnet-10

      the repo ships kascov-lab — a covenant lifecycle lab that births, moves, and retires a real covenant on testnet-10 with faucet funds. launch one, then paste your txid into the search box here and watch kascov tell its story.

      cargo run -rp kascov-lab -- --help
