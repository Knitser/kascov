      all smart coins

      script decoder

      paste any Kaspa script as hex and read it as opcodes — including the post-Toccata set: transaction introspection (KIP-17), covenant ops (KIP-20), zk verification (KIP-16). decoding happens in your browser; nothing leaves the page.

      what the colors mean
      • push data pushed onto the stack
      • standard the classic script opcodes (checks, math, stack)
      • introspection KIP-17 — the script reads its own transaction
      • covenant KIP-20 — smart-coin identity & state rules
      • zk KIP-16 — zero-knowledge proof verification
      • unknown not in the post-Toccata opcode table

      for developers

      kascov indexes every covenant on Kaspa and keeps the history nodes prune. all of it is yours as plain JSON — CORS is on, no keys, no rate cards. build something.

      the JSON API

      GET /data/testnet-10-live.json

      the small fast feed — stats, chain tip, and the ~150 newest life events. refreshes every few seconds; poll this.

      GET /data/testnet-10.json

      the full snapshot — every covenant with its complete event timeline and UTXOs (scripts included, pre-disassembled). a few MB; fetch it when the live feed says something changed.

      GET /data/mainnet.json · /data/mainnet-live.json

      same, for mainnet — mostly waiting for its first smart coins.

      base URL: https://kascov-explorer.web.app · Access-Control-Allow-Origin: * · served by an always-on indexer that follows the chain live

      try it

      # the pulse of the testnet, one request
curl -s https://kascov-explorer.web.app/data/testnet-10-live.json | jq .stats

# coins born in the last few minutes
curl -s https://kascov-explorer.web.app/data/testnet-10-live.json \
  | jq '.recent_events[] | select(.kind == "genesis")'

# one covenant's whole life
curl -s https://kascov-explorer.web.app/data/testnet-10.json \
  | jq --arg id "<covenant-id>" '.covenants[] | select(.covenant_id == $id)'

      field dictionary

      covenant_id
      the smart coin's permanent 32-byte identity (KIP-20) — survives every move
      status
      active (has live UTXOs) or burned (retired)
      events[]
      the life story: kind is genesis | transition | burn, ordered by seq; txid is the transaction, accepting_daa when the chain accepted it
      accepting_daa / DAA
      Kaspa's block-height-like clock — it ticks ~10 per second. tip_daa + tip_at_ms in each snapshot anchor it to wall-clock time
      genesis_txid, genesis_daa
      where the coin was born — null if it was first seen mid-life (see lineage_complete)
      lineage_complete
      false means kascov started watching after the coin was born; earlier history is honestly missing
      utxos[]
      every state UTXO ever seen: outpoint, value (sompi, 1e8 = 1 KAS), live, script_hex + script_asm (pre-disassembled), uses_covenant_ops, uses_zk_ops
      live_value, live_utxos
      what the coin holds right now
      events_truncated
      the full snapshot caps events per covenant (default 500); true means the tail was cut
      generated_at_ms
      when this JSON was built — your freshness check

      run kascov yourself

      the whole indexer is a small open-source Rust workspace. your own node, your own index, your own rules.

      git clone https://github.com/Knitser/kascov && cd kascov

# follow testnet-10 live (public resolver, zero setup)
cargo run -rp kascov -- --network testnet-10 sync --follow

# print covenant events as they happen (great for testing your own covenant)
cargo run -rp kascov -- --network testnet-10 watch

# machine-readable event stream
cargo run -rp kascov -- --network testnet-10 --json watch

# export the snapshots this site serves
cargo run -rp kascov -- --network testnet-10 export

      make your own smart coin on testnet-10

      the repo ships kascov-lab — a covenant lifecycle lab that births, moves, and retires a real covenant on testnet-10 with faucet funds. launch one, then paste your txid into the search box here and watch kascov tell its story.

      cargo run -rp kascov-lab -- --help
